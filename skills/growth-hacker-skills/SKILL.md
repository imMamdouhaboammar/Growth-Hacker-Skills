# Growth Hacker Skills

## Role

This is the Claude plugin entry for the Growth Hacker Skills network created and maintained by Mamdouh Aboammar.

The canonical router is the repository-level `SKILL.md`.

Before coordinating a multi-skill workflow, read:

1. `../../SKILL.md`
2. `../../skill-graph.json`
3. `../../docs/neural-linking.md`
4. `../../templates/growth-context.md`

Then follow the canonical router without replacing its routing, handoff, context, or quality rules.

## Fast routing

- Missing voice foundation: `voice-builder`
- Fresh niche evidence: `niche-research`

- Structured ideation: `content-matrix`

- Named copy framework: `post-formatter`

- Pinned community comment: `pinned-comment`
- Quote-led asset: `quote-post`
- Publishable LinkedIn quality review: `post-scorer`

## Neural handoff

For workflows using more than one specialist skill:

1. Resolve the path from `../../skill-graph.json`.
2. Create or update `.growth-hacker/context.md` when file tools exist.
3. Carry approved objective, audience, evidence, angle, voice, brand, hook, format, CTA, completed nodes, and risks.
4. Run one node at a time.
5. Stop when the requested deliverables are complete.

Never make a receiving skill repeat completed discovery.

Never exceed five specialist hops without explicit user approval.

Never convert a source claim or inference into a verified fact.

Never use an em dash in generated copy.
