# ChatGPT Images Prompt Template

Use this template for both Safe and Experimental directions. Replace every bracketed field before output. The final prompt contains no placeholders.

```text
Create one polished editorial social graphic for ChatGPT Images.

OUTPUT
- Canvas: [1080 x 1350 pixels, 4:5 vertical, or approved alternative]
- Language: English only
- Purpose: [educational, announcement, system map, comparison, guide, promotional, commentary]
- Audience: [specific audience]

SOURCE FIDELITY
Base the design only on the supplied source analysis. Do not add features, statistics, dates, pricing, logos, interfaces, integrations, testimonials, or claims that are not in the source. Treat attributed source claims as attributed claims. Do not imply independent verification.

CREATIVE DIRECTION
- Angle: [chosen angle]
- Format: [chosen format]
- Reference grammar: [style ID and explicit visual grammar]
- Visual concept: [one-sentence composition idea]
- Information density: [low, medium, medium-high, high]

EXACT ON-IMAGE COPY
Render the following text exactly as written. Do not paraphrase, correct, expand, shorten, reorder, duplicate, or add text.

Headline:
"[exact headline]"

Deck:
"[exact deck or omit this section]"

Section 1 label:
"[exact label]"
Section 1 support:
"[exact support]"

[Repeat exact content units]

Footer:
"[exact footer or omit]"

TYPOGRAPHY
- Headline font: [font and weight]
- Body font: [font and weight]
- Technical labels or code font: [font and weight, when relevant]
- Use clear Latin letterforms and preserve spelling, punctuation, numbers, capitalisation, and product names.
- Establish strong size contrast between headline, deck, labels, support text, and footer.

LAYOUT
- [Exact grid, zones, proportions, alignment, whitespace, card positions, connector paths, and reading order]
- Keep a minimum safe margin of [value] around all edges.
- Keep text-bearing panels front-facing and undistorted.
- Make the full hierarchy readable on a mobile feed.

PALETTE AND MATERIAL
- Background: [colour and treatment]
- Primary text: [colour]
- Accent 1: [colour and purpose]
- Accent 2: [colour and purpose]
- Lines and borders: [colour and weight]
- Texture: [none, grid, paper, subtle glow, terminal grain]

VISUAL ELEMENTS
- [Specific diagrams, icons, screenshots, cards, arrows, timelines, or neutral illustrations]
- Use verified source assets only when they were supplied.
- When no verified product screenshot exists, use neutral editorial diagrams instead of fictional UI.

NEGATIVE CONSTRAINTS
- No misspelled text
- No extra text
- No fake logos
- No fictional product screenshots
- No invented features or metrics
- No random icons
- No glossy generic AI 3D
- No stock-photo look
- No excessive gradients
- No decorative clutter
- No tiny unreadable paragraphs
- No warped cards or perspective-distorted text
- No copied creator identity from the references

FINAL QUALITY
The result should feel designed by a senior editorial information designer. Prioritise clarity, source accuracy, hierarchy, and mobile readability over decoration.
```

## Safe direction

Use the reference grammar directly and keep the layout conventional, balanced, and highly readable.

## Experimental direction

Keep the exact source facts and copy, but vary one or more of:

- Composition rhythm
- Cropping
- Layering
- Diagram shape
- Negative space
- Editorial metaphor
- Type scale contrast

Do not experiment with factual content or text wording.
