# Editorial Visual Style Library

The permanent reference board is `assets/reference-board.svg`. Use the style IDs below. Reuse visual grammar, not exact compositions, text, identities, or creator branding.

## EV-01. Benchmark Field Guide

Visual grammar:

- White or warm-white background with a faint square grid
- Strong navy headline with one coral keyword
- Top row contains comparison metrics and a timeline
- Lower area uses stacked numbered use-case cards
- Small interface or icon vignette at the right of each card
- Dark footer strip for author identity

Best for:

- Model comparisons
- Product availability windows
- Ranked use cases
- Launch guides with dates and limits

Density: high but orderly.

## EV-02. Circular Multi-Agent Map

Visual grammar:

- White background
- Central product or output mockup
- Dotted circular path
- Five or six agent nodes around the centre
- Purple role pills, orange agent icons, black headings
- Short descriptions outside the orbit

Best for:

- Agent workflows
- Multi-stage production systems
- Repository architecture
- Team roles around one deliverable

Density: medium.

## EV-03. Launch Pipeline Poster

Visual grammar:

- White background with restrained warm glow
- Large black headline with orange announcement phrase
- Prompt or chat card near the upper centre
- Horizontal five-stage process with distinct accent colours
- Output examples or channel mockups along the bottom
- Founder or offer CTA card at the footer

Best for:

- Product launches
- AI skill announcements
- Creative production workflows
- Free resource offers

Density: medium-high.

## EV-04. Dark GTM Architecture

Visual grammar:

- Near-black background with dark brown or orange radial glow
- Orange outlines and connectors
- Central integration block at top
- Modular workflow cards below
- Dotted architecture lines
- Two dense explanatory panels at the bottom

Best for:

- GTM systems
- Technical stacks
- Enrichment and outbound workflows
- Integration-heavy tools

Density: high.

## EV-05. Paper Step Playbook

Visual grammar:

- Light paper texture
- Large black serif headline with one terracotta phrase
- Seven stacked step cards
- Dark labels at left
- Old versus new contrast markers
- Code or prompt snippets inside coloured boxes
- Final step uses a dark inverted panel

Best for:

- Setup guides
- Tutorials
- Migration checklists
- Old process versus new process

Density: high.

## EV-06. Advisor Board Synthesis

Visual grammar:

- White grid background
- Navy and coral typography
- Four-step setup strip
- Mentor or role cards across the middle
- Coloured convergence lines into a central synthesiser
- Large verdict panel at the bottom
- Small command chips at the footer

Best for:

- Multi-perspective analysis
- Decision frameworks
- Advisory boards
- Synthesis workflows

Density: medium-high.

## EV-07. Minimal Terminal Poster

Visual grammar:

- Near-black terminal window
- Monospace typography
- Two large product marks separated by a plus sign
- Lime or green CTA box
- Sparse composition with heavy negative space
- Small terminal prompt or domain at the bottom

Best for:

- Integration announcements
- API guides
- Developer tools
- One-message offers

Density: low.

## EV-08. Dense Company Org Chart

Visual grammar:

- Deep navy background with a subtle grid
- One central operating node at the top
- Dotted connectors to colour-coded departments
- Repeated card system for many skills or tools
- Small mascots or icons used consistently
- Footer identity strip

Best for:

- Skill libraries
- Department maps
- Large agent collections
- Capability inventories

Density: very high. Use only when the source genuinely contains many grouped items.

## Compatible mixes

Allowed pairs:

- EV-01 + EV-05 for a benchmark that becomes a tutorial
- EV-02 + EV-06 for multi-agent synthesis
- EV-03 + EV-01 for a launch plus use-case guide
- EV-04 + EV-07 for a dark architecture with a minimal CTA
- EV-08 + EV-02 for a large library with one central operating loop

Avoid mixing:

- EV-05 paper texture with EV-04 dark glow
- EV-07 minimalism with EV-08 dense inventories
- More than two style IDs

## Style selection rule

Choose by information structure first, mood second. A visually attractive style is wrong when it hides the source's structure.
