# English Typography Library

## Default rule

Use English copy and fonts with reliable Latin character support. Never select a decorative font for body copy or code.

## Recommended pairings

### Editorial authority

- Headline: Helvetica Neue Condensed Bold or Helvetica Neue Bold
- Body: Inter Regular and Inter Medium
- Metadata: Inter Medium in uppercase with tracking
- Best for: field guides, benchmarks, timelines, comparison boards

Fallback when Helvetica Neue is unavailable: Inter Tight Bold + Inter.

### Technical system

- Headline: IBM Plex Sans Bold
- Body: IBM Plex Sans Regular
- Code and labels: IBM Plex Mono Medium
- Best for: repositories, agent maps, API workflows, terminal posters, system diagrams

### Premium modern editorial

- Headline: Neue Montreal Bold
- Body: Inter Regular
- Labels: Neue Montreal Medium in uppercase
- Best for: launch graphics, founder-led posts, clean product diagrams

Fallback when Neue Montreal is unavailable: Manrope ExtraBold + Inter.

### Paper playbook

- Headline: Instrument Serif Bold or Georgia Bold
- Body: Inter or Source Sans 3
- Code: IBM Plex Mono
- Best for: step-by-step guides, old versus new comparisons, editorial manuals

## Sizing hierarchy for 1080 x 1350

Use relative proportions, not a rigid export assumption:

- Headline: 7 to 10 percent of canvas width in perceived size
- Deck: 3 to 4 percent
- Section label: 2.2 to 3 percent
- Body: 2 to 2.7 percent
- Footer: 1.5 to 2 percent

## Text reliability rules

- Use no more than three font families in one graphic
- Prefer two families
- Use one weight change before adding another colour
- Keep body copy left aligned in English layouts unless the visual system requires centred labels
- Avoid thin weights below 500 for small reversed text
- Use monospace only for code, prompts, terminal labels, or technical metadata
- Use uppercase sparingly for labels, not paragraphs
