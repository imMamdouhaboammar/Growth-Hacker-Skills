# Example: Repository to Editorial System Map

## User request

> Analyse this repository and create the full editorial visual pack. Use ChatGPT Images. Include LinkedIn and X captions plus a Google Flow prompt.

## Expected route

1. Read repository README, docs, package metadata, examples, and relevant core files.
2. Mark access `FULL` or `PARTIAL` honestly.
3. Separate verified capabilities from README marketing claims.
4. Generate source-specific angles such as:
   - The repository architecture in one map
   - Five practical workflows inside the project
   - What changed in the latest release
5. Recommend EV-02, EV-04, or EV-08 based on the information structure.
6. Ask all creative choices in one batch.
7. Produce Safe and Experimental ChatGPT Images prompts.
8. Produce Google Flow motion only when selected.
9. Apply voice.md to captions when available.

## Failure conditions

- Reading only the README while claiming full repository analysis
- Inventing integrations from logos
- Showing fictional product UI
- Adding external benchmark data in source-only mode
